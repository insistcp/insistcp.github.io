Tensorflow学习笔记-第二讲Tensorflow实现自编码和多层感知机

发表于 2017-05-31 | | 阅读次数

一.问题描述

传统的机器学习依赖于特征工程，因为我们在实施我们的算法时往往都是利用特征量。但是特征的提取往往耗时耗力，并且部分特征提取需要相对专业的人士才可以进行。深度学习则可以通过算法解决这个繁琐的特征提取过程。深度学习模仿人脑的逐层抽象的过程，从最最简单的概念到更高层抽象的概念：其中最重要两点：

  • 1：无监督学习，即我们不需要标注数据就可以对数据组织形式进行一定程度的学习，
  • 2：逐层抽象，特征是需要不断抽象的。

例如：一张图片里面的汽车，我们要怎么样将其识别出来，计算机开始拥有的是一个个像素数字，我们不可能直接通过这些像素数字将内容识别出来，所以我们可以将这些像素进行组合抽象成更高阶的特征，比如汽车的轮胎，车门等。即特征是可以不断被转化为更高阶更加抽象的特征，问题来了？我们如何找到找到这些基本结构呢？使用无监督的自编码提取特征，
自编码：一种简单的神经网络，它的输入和输出是一致的，借助稀疏编码的思想，目标是使用稀疏的一下高阶特征来重构自己。

四.利用tensorflow实现多层感知机

由于Softmax　regression模型拟合能力不够，于是我们使用多层神经网络模型来解决手写数字识别l；多层感知机相比于SoftMax Regression而言多了一层隐藏层（就是我们常说的神经网络的黑盒子，一般而言对外界是不可见的），（PS：理论上只要隐藏层的节点足够多，即使只有一个隐藏层我们也可以拟合十分复杂的函数）；同时隐藏层越多越容易拟合复杂的函数。有理论表明，隐藏层的层数越多，隐藏层需要的节点数就越少。这就是深度学习的特点之一：层数越深，概念越抽象，需要背诵的知识点就越少。但是随着层数的增加，一些问题逐渐显现出来，例如：过拟合，参数难以调试，梯度弥散等好的是近几年对于出现的各种问题，人们都想出了对应的解决办法

  • 1：过拟合：使用Dropout，将神经网络的某一层的输入节点数据随即丢弃一部分。本质上是创造出来更多的新的随机样本，通过增大样本量减少特征数量来防止过拟合。
  • 2：参数难以调试，尤其是对于SGD算法而言，采用不同的学习率，得到的结果可能截然不同。因此提出了Adagrad，Adam,Adadelta等调节参数的算法，这些算法使得通常我们给定一个默认的参数就会得到一个比较好的效果。
  • 3：梯度弥散：
    • 1.函数饱和使梯度消失
      sigmoid 神经元在值为 0 或 1 的时候接近饱和，这些区域，梯度几乎为 0。因此在反向传播时，这个局部梯度会与整个代价函数关于该单元输出的梯度相乘，结果也会接近为 0 。这样，几乎就没有信号通过神经元传到权重再到数据了，因此这时梯度就对模型的更新没有任何贡献。除此之外，为了防止饱和，必须对于权重矩阵的初始化特别留意。比如，如果初始化权重过大，那么大多数神经元将会饱和，导致网络就几乎不学习。
    • 2.sigmoid 函数不是关于原点中心对称的
      这个特性会导致后面网络层的输入也不是零中心的，进而影响梯度下降的运作。因为如果输入都是正数的话（如 f=wTx+b 中每个元素都x>0 ），那么关于 w的梯度在反向传播过程中，要么全是正数，要么全是负数（具体依据整个表达式 f而定），这将会导致梯度下降权重更新时出现 z 字型的下降。当然，如果是按 batch 去训练，那么每个 batch 可能得到不同的信号，整个批量的梯度加起来后可以缓解这个问题。因此，该问题相对于上面的神经元饱和问题来说只是个小麻烦，没有那么严重。

解决办法采用ReLU作为激活函数
近些年来非常流行。它的数学公式为：f(x)=max(0,x)。

  • 优点：

    • 1:相较于 sigmoid 和 tanh 函数，ReLU 对于 SGD 的收敛有巨大的加速作用（Alex Krizhevsky 指出有 6 倍之多）。有人认为这是由它的线性、非饱和的公式导致的。
    • 2:相比于 sigmoid/tanh，ReLU 只需要一个阈值就可以得到激活值，而不用去算一大堆复杂的（指数）运算。

  • 缺点
    ReLU 的缺点是，它在训练时比较脆弱并且可能“死掉”。
    举例来说：一个非常大的梯度经过一个 ReLU 神经元，更新过参数之后，这个神经元再也不会对任何数据有激活现象了。如果这种情况发生，那么从此所有流过这个神经元的梯度将都变成 0。
    也就是说，这个 ReLU 单元在训练中将不可逆转的死亡，导致了数据多样化的丢失。实际中，如果学习率设置得太高，可能会发现网络中 40% 的神经元都会死掉（在整个训练集中这些神经元都不会被激活）。
    合理设置学习率，会降低这种情况的发生概率。
